一、css-loader的使用

1、在官网文档中找到css-loader配置

2、安装：npm install --save-dev css-loader 解析并加载CSS文件

3、安装：npm install style-loader --save-dev 将导出的样式加到DOM中

4、导入css文件 import css from 'file.css';

5、在webpack.config.js中配置module

6、配置package.json中的build选项

7、执行打包：node run build

二、打包.less文件

1、安装 less-loader 和 less：npm install --save-dev less-loader less

### less-loader用于加载，less用于将less文件转化成css

### 这里我安装失败了，原因webpack版本低，升级webpack或者选择低版本的less-loader

### 我选择了升级webpack到最新版本：cnpm install --save-dev webpack

### 同时升级到4.0后执行webpack提示安装webpack-cli（npm install -D webpack-cli），原因4.0以后需要cli

2、配置webpack.config.js

3、导入less文件

4、执行打包： npm run build

三、打包背景图

1、添加css样式

2、安装url-loader：npm install --save-dev url-loader

3、配置webpack.config.js

4、此时执行npm run build 会报错缺少file-loader

### 小于limit（在webpack.config.js中配置）的文件的url会直接转成base64字符串传输， 大于limit的使用file形式传输，所以这里需要安装file-loader，只安装即可，无需其他配置

### 执行安装：cnpm install --save-dev file-loader

### 此时大于limit的背景图报ERR_FILE_NOT_FOUND，原因是打包到dist后路径改变了，文件名也变成了hash值， 此时需要配置打包的文件路径及文件文件名规则

### 文件名规则：name: 'img/[name].[hash:8].[ext]'，[hash:8]代表8位hash

### 在webpack.config.js中配置资源的查找路径： publicPath: 'dist/'
